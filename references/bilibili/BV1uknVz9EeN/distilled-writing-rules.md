# BV1uknVz9EeN：剧本写作蒸馏规则

> 证据为 0–180 秒硬字幕 OCR。以下规则学习的是人物语言机制，不确认 OCR 中每一位说话者的身份，也不把具体台词当可复制模板。

## 1. 角色声音观察

本片可分出四种“功能声部”，它们不必等同于具体角色姓名：

- **催促/行动声部**：句子短，先叫名字，再报眼前状态，再换办法；越急越重复，不会一次说明完。
- **游离/梦话声部**：对问题的回答不接题，突然转向食物、出国或其他具体欲望；它不是“逻辑错误”，而是当下注意力不在同一件事上。
- **过度执行声部**：把群体的半开玩笑建议理解成必须立刻实施的方案，语气直接、行动词多。
- **现实质疑声部**：不急着接情绪，先问“具体怎么回事”“这玩意有用吗”，或校正细节；这会让群像不全都沿同一条情绪线冲刺。

名称、称呼和省略号本身就是声音的一部分：反复叫人不是信息冗余，而是关系与焦急的节拍。

## 2. 对话节奏规则

1. **共同问题不必得到共同回答。**
   - 观察现象：围绕“叫不醒”这一个事件，回应先后落到学习、留学、食物、茶和物理唤醒上。
   - 执行方法：每个角色只回应自己最在意的一个切口；让后一人接歪前一句，不要轮流复述问题。
   - 适用：熟人群像、轻喜剧、焦虑被打断的场景。
   - 滥用风险：所有人都接歪会让主线消失；每 2–4 句应有人把场面拉回现实。
   - 证据时间点：00:01:21–00:02:32。
   - 可信度：★★★★☆

2. **把升级拆成小台阶，而不是一口气喊到最高。**
   - 观察现象：叫名字 → 催起床 → 换理由 → “坏了” → “紧急消息”。
   - 执行方法：每次只增加半级行动或措辞；前一招失效后再露出下一招。
   - 适用：担心、寻找、劝说、闹剧。
   - 滥用风险：每句升级会变成刻意吵闹。
   - 证据时间点：00:00:10–00:01:48。
   - 可信度：★★★★☆

3. **允许句子在真正想说完之前转弯。**
   - 观察现象：多处出现“那……”“就是……”“不对……”及句末截断。
   - 执行方法：在角色改主意、心虚或被人插话的位置停下；下一句不必补全原句。
   - 适用：临场反应、关系敏感话题。
   - 滥用风险：所有人都省略会难读；给每个角色不同的断句习惯。
   - 证据时间点：00:00:47–00:00:52；00:02:16–00:02:22。
   - 可信度：★★★★☆

## 3. 情绪表达规则

4. **焦急先表现为重复称呼和操作，不要先命名情绪。**
   - 观察现象：焦急通过连续叫人、摇人、发消息和不断试新办法出现；“着急”不是主要台词内容。
   - 执行方法：把“我很担心”替换成一次未成功的动作、一个重复称呼和一个更不靠谱的新方案。
   - 适用人物：行动快、关心却不善于表白的人。
   - 不适用：需要明确交代风险的严肃场景。
   - 滥用风险：只做动作会让人物显得无感；保留一两句失控的短判断。
   - 证据时间点：00:00:10–00:00:43；00:01:28–00:01:48。
   - 可信度：★★★★★

5. **嘴硬/游离的情绪，用具体物件泄露，不用心理独白解释。**
   - 观察现象：面对唤醒和危机，回应却落在草莓三明治、炸鱼薯条、茶等物件上。
   - 执行方法：让人物回避核心问题时抓住一个过于具体的需求、菜名、工具或无关细节；读者从偏题感读出其状态。
   - 适用人物：困倦、心虚、逃避、习惯转移话题的人。
   - 不适用：信息必须被清楚传达的剧情节点。
   - 滥用风险：物件不能随机；要和该角色的欲望或当下情境有关。
   - 证据时间点：00:00:19–00:00:22；00:01:21–00:01:28；00:02:25–00:02:32。
   - 可信度：★★★★☆

6. **夸张不是情绪直给，而是把方案推进得过头。**
   - 观察现象：有人从“叫醒”直接跳到过激、字面执行的处理方案，旁人再以现实问题或更荒诞的误解接住。
   - 执行方法：让角色把一句半真半假的建议执行到不合时宜；再让另一人只纠正一个细节，不总结道德。
   - 适用：熟人互损、轻喜剧、情绪过载后的泄压。
   - 不适用：真实暴力、创伤或需要严肃处理的情境。
   - 滥用风险：必须清楚是虚构夸张和关系许可，不能让伤害本身成为笑点。
   - 证据时间点：00:01:48–00:02:01；00:02:42–00:02:57。
   - 可信度：★★★☆☆

## 4. 人物关系驱动的语言变化

7. **熟人关心可以先以不客气的催促出现。**
   - 观察现象：昵称、直呼、催促、吐槽和接梗同时存在；关心不被包装成温柔完整句。
   - 执行方法：关系亲近时，允许“叫名 + 短命令 + 失败后的再试”；关系生疏时改为事实询问和留出选择权。
   - 适用关系：亲近同伴、共同体内部。
   - 不适用：关系未建立或存在权力压迫时。
   - 滥用风险：不要把粗鲁当亲密；要有对方能回嘴、能拒绝的证据。
   - 证据时间点：00:00:10–00:00:43；00:02:07–00:02:15。
   - 可信度：★★★★☆

8. **冲突中保留一个“细节校正者”。**
   - 观察现象：群体越说越离谱时，有人不说“你们别闹了”，而是问具体状态，或指出工具的用途不对。
   - 执行方法：给理性角色一句小而准的校正；不要让他发表完整说教。
   - 适用关系：多人聊天、轻度冲突、计划失控。
   - 不适用：需要角色彻底翻脸的场景。
   - 滥用风险：校正者不能永远正确，否则会变作者嘴替。
   - 证据时间点：00:02:07–00:02:15；00:02:22–00:02:32。
   - 可信度：★★★★☆

9. **称呼变化是最省字的关系镜头。**
   - 观察现象：昵称、带后缀的称呼、直接点名和“小 + 名字”承担了距离、提醒与劝阻。
   - 执行方法：为每组关系预设 2–3 种称呼；情绪升高时切换一种，而不是加长解释。
   - 适用：群像、朋友关系的升温或紧张。
   - 滥用风险：不要每句都叫全名；称呼切换必须有触发条件。
   - 证据时间点：00:00:10–00:00:52；00:02:07–00:02:57。
   - 可信度：★★★★☆

## 5. 可直接用于剧本的反 AI 规则

10. **避免完整解释型台词 → 改成“行动 + 半句 + 下一招”。**
   - 原因：真人着急时通常先处理眼前事，话会被行动和失败切断。
   - 写法：不要写“我担心你无法醒来，所以我联系大家”。改为先叫人、试一次、说“坏了”，再去发消息。
   - 证据时间点：00:01:28–00:01:48。
   - 可信度：★★★★★

11. **避免每句都正面回答 → 给每个角色一个偏题方向。**
   - 原因：不同人物有不同注意力和防御方式，偏题会暴露人而不是只推进情节。
   - 写法：一人抓后果，一人抓物件，一人抓字面方案，一人抓事实核验。
   - 证据时间点：00:01:21–00:02:32。
   - 可信度：★★★★★

12. **避免用“我很生气/很担心/很无语”作结 → 用反应延迟和称呼收句。**
   - 原因：情绪从停顿、重复、突然缩短的评价里更可见。
   - 写法：保留“啊……”“等一下”“你是否清醒”这一类短反应；让读者补出情绪。
   - 证据时间点：00:01:53–00:02:07；00:02:33–00:03:00。
   - 可信度：★★★★☆

13. **避免所有角色都逻辑清楚、语气礼貌 → 分配不同的表达缺口。**
   - 原因：群像的辨识度来自“谁会省略、谁会夸张、谁会纠正、谁会接错”。
   - 写法：写前先给每人标一个默认偏差，并在同一场对话里至少体现两种。
   - 证据时间点：00:01:37–00:02:57。
   - 可信度：★★★★★

14. **避免为了推进剧情而齐声升级 → 用一条拉回现实的短句制造反差。**
   - 原因：全员同向会像作者安排；错位校正会留下人际摩擦。
   - 写法：当两人夸张升级后，第三人只问一个具体问题或纠正一个用途。
   - 证据时间点：00:02:07–00:02:32。
   - 可信度：★★★★☆

## 6. 可保留的社区共识表达

以下只建议保留为短促、低频、关系明确的点缀：

- **带后缀的昵称/内部称呼**：它的作用不是卖萌，而是迅速标记谁在对谁说、关系有多近。建议在转场、劝阻、失控前使用，不要每句重复全称。
- **“坏了”式短判断**：比长篇解释更快地把焦急推到下一步；有效处在于它留出“到底坏在哪”的空白。
- **“等一下”式切断**：用于把一条过火方案截住，能显示关系中有人敢打断，也有人愿意被打断。
- **“这玩意”式具体物件代称**：可保留口语感和临场感；前提是观众已看见或刚被提到该物件。
- **夸张数量词/重复前缀**：只作一两拍的失控信号，之后应被自我修正或他人吐槽接住。

不要整段复刻本片的连续笑点、特定专名串联或完整高潮结构；社区共识的价值在“读者一眼懂得关系”，不是把梗当对白主体。

## 7. 最终写作清单

- [ ] 这个角色的句长、停顿和回应角度，是否明显不像另一个角色？
- [ ] 是否至少有一个人没有正面回答，而是以符合自身习惯的方式偏题？
- [ ] 是否把“担心/生气/尴尬”改写成了称呼、动作、短判断或具体物件？
- [ ] 有没有把本应断掉的半句硬补成说明书式完整句？
- [ ] 夸张升级后，是否有一个小而具体的现实校正，而非作者说教？
- [ ] 称呼变化、句子变短、停顿变多，是否随关系和压力变化？
- [ ] 每个笑点或冲突，是否仍由人物各自的偏差推动，而不是为了推进剧情而说？
- [ ] 是否只借用了语言机制，而没有机械拼接原片连续台词、完整笑点或社区梗？
