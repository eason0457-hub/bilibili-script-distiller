# 单视频字幕整理与蒸馏规则

## 1. 原始字幕

`subtitle-raw.md` 必须保留原顺序与时间戳，不文学润色、不静默纠错。疑似错误写成旁注，如 `<!-- 疑似识别错误：... -->`，原文仍保留。

## 2. 可读字幕

允许补基础标点、合并时间轴切碎的同一句、按有证据的说话者分段、删除完全重复的技术字幕。必须保留停顿、重复、语气词、口头禅、半句话、自我修正、答非所问、被打断的话、未获回答的问题和不完整口语。不得把人物改得更聪明、完整、礼貌或理性。

说话者不确定时使用 `说话者A`、`说话者B`、`未知说话者`，不得凭声线印象或剧情常识猜身份。

## 3. 分析维度

对每位主要说话者分别记录：

- 平均句长及压力下的长短句变化；
- 停顿位置、句尾、语气词、自称、称呼；
- 重复、自我修正、省略、打断与是否直接回答；
- 回避、答非所问、心虚时过度解释等具体路径；
- 生气、害怕、被关心时的语言变形；
- 面对不同关系对象时的称呼、完整度与解释权变化；
- 潜台词，以及动作、物件、环境与对白的配合；
- 冲突如何开始、升高、暂停、回落或悬而未决；
- 能避免公式化对白的可复现机制。

禁止只用“自然、细腻、鲜明、温柔、冷静、傲娇、内向、有张力”等形容词作结论。每个判断必须指向具体语言现象和时间点。

## 4. 可执行规则格式

`distilled-writing-rules.md` 至少分为：人物专属语言、关系型语言、场景型语言、对话节奏、描写与旁白、去 AI 味机制、社区共识表达。

每条规则使用：

```markdown
### 规则名称：
观察现象：
执行方法：
适用人物：
适用关系：
适用场景：
不适用情况：
滥用风险：
证据时间点：
可信度：
```

可信度：`★★★★★` 多个片段稳定出现；`★★★★☆` 多次出现但依赖关系或情绪；`★★★☆☆` 特定场景有效；`★★☆☆☆` 单个片段，谨慎使用；`★☆☆☆☆` 特殊表达或社区梗，仅作点缀。

## 5. 原作语言边界

可直接保留固定自称、固定称呼、简短口癖、固定句尾、反复出现的简短反应、社区共识短梗、角色识别短句型和组织内部固定称谓。逐条标记使用人物、对象、场景、建议频率、滥用风险与来源时间点。

优先迁移表达机制：先否认再泄露需求；回答字面问题而回避真正问题；重复关键词反击；纠正无关细节后回避核心；心虚时解释过多；生气时敬语增加或句子缩短；靠近他人前先谈小问题；用具体物件代替情绪回答；以称呼变化表现关系变化。

不得默认复制连续多句完整对白、完整高潮场景、完整笑点结构或长篇独特比喻。提取机制并给出原创短例，避免重现原场景的对象、信息顺序和落点。
